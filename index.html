HTML Nedir?

HTML (HyperText Markup Language) web sayfalarının iskeletidir. İçeriği tanımlar ama görünümü değil. Görünüm CSS'in işi.

HTML — Temel Sayfa Yapısı 
<!DOCTYPE html>          ← Tarayıcıya "HTML5 kullanıyorum" der
<html lang="tr">            ← Kök eleman, dil belirt
  <head>                      ← Görünmez meta bilgiler
    <meta charset="UTF-8">
    <title>Sayfam</title>
  </head>
  <body>                      ← Görünen her şey burada
    <h1>Merhaba Dünya!</h1>
  </body>
</html>
Altın Kural: HTML = yapı, CSS = görünüm, JavaScript = davranış. Bu ayrımı asla karıştırma.

HTML Temelleri

Semantic HTML

Semantic = anlamlı. Her şeyi div yapmak yerine, içeriği tanımlayan etiketleri kullanmak hem okunabilirliği hem SEO'yu iyileştirir.

❌ Kötü (amatör):
Her şey div
HTML — Yanlış
<div class="header">...</div>
<div class="nav">...</div>
<div class="content">...</div>
<div class="footer">...</div>
✅ İyi (pro):
Anlamlı etiketler
HTML — Doğru
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

Semantic Etiket Haritası

<header> — Logo, site başlığı, üst menü
<nav> — Navigasyon linkleri
<main>
<section> — Bölümler
<article> — Bağımsız içerik
<aside>
Yan panel
<footer> — Alt bilgi, telif, iletişim
Pro Trick: Semantic etiketler hem SEO'yu (arama motoru sıralaması) hem de erişilebilirliği (ekran okuyucular) iyileştirir. <header>, <nav>, <main>, <footer> kullan — kodunu okuyan herkes yapıyı anında kavrar.

Ne Zaman Hangisini Kullanırsın?

<header> Sayfanın veya bir bölümün üst kısmı. Logo, site adı, üst menü genellikle burada.
<nav> Navigasyon linkleri içindir. Menü linklerini buraya koy, düz link listelerini değil.
<main> Sayfanın ana içeriği. Sayfada sadece bir tane olmalı. Tekrarlanan içerikler (nav, footer) buraya girmez.
<section> Tematik olarak gruplanmış içerik bölümü. Başlığı olan her büyük alan section olabilir.
<article> Bağımsız, kendi başına anlam taşıyan içerik. Blog yazısı, haber, yorum gibi. Sayfadan koparıp başka yerde de yayınlanabilir.
<aside> Ana içerikle ilgili ama yan bilgi. Sidebar, reklamlar, ilgili linkler.
<footer> Sayfanın veya bir bölümün alt kısmı. Telif hakkı, iletişim, sosyal medya linkleri.

Form Yapıları

Formlar kullanıcıdan veri toplayan en temel araç. Her input bir label ile eşleşmeli.

HTML — Tam Form Örneği
<form action="/gonder" method="POST">

  
  <label for="isim">Adınız</label>
  <input type="text" id="isim" name="isim" placeholder="Adınızı girin">

  <label for="email">E-posta</label>
  <input type="email" id="email" required>

  <label for="mesaj">Mesaj</label>
  <textarea id="mesaj" rows="4"></textarea>

  <button type="submit">Gönder</button>
</form>

Pro Trick: label for="id" ve input id="id" eşleşmesi şart. Bu olmadan erişilebilirlik bozulur — label'a tıkladığında input odaklanmaz, ekran okuyucular formu anlayamaz.

Canlı Form Önizlemesi

Gerçekte nasıl görüneceği — tıklayabilir ve doldurabilirsin:

Form Nitelikleri (Attributes)

action="/url" Form verisinin hangi adrese gönderileceği. Backend URL'i buraya yazılır.
method="POST" POST = veriyi body'de gönder (güvenli, form için tercih et). GET = URL'de görünür, arama kutuları için.
required Bu alan doldurulmadan form gönderilemez. Tarayıcı otomatik hata mesajı gösterir.
placeholder Input boşken görünen açıklama metni. Kullanıcı yazmaya başlayınca kaybolur.
name="..." Verinin sunucuya gönderilirken hangi anahtar ile gönderileceği. Backend için zorunlu.

CSS Nedir?

CSS (Cascading Style Sheets) HTML iskeletine görünüm kazandırır. Renk, boyut, konum, animasyon — hepsi CSS.

CSS Nasıl Eklenir?

❌ INLINE — kullanma
<p style="color:red">Metin</p>
⚠️ INTERNAL — küçük projeler
<style>
  p { color: red; }
</style>
✅ EXTERNAL — her zaman bu
<link rel="stylesheet" href="style.css">

Selector'lar

CSS hangi HTML elemanını hedefleyeceğini selector'larla bilir.

CSS — Temel Selector'lar
p { }               ← Tüm p etiketleri
.kart { }          ← class="kart" olan elemanlar
#logo { }          ← id="logo" olan TEK eleman
* { }               ← Her şey (evrensel)

/* Kombinasyonlar */
div p { }          ← div içindeki TÜM p'ler
div > p { }        ← div'in DOĞRUDAN çocuğu p
h1, h2 { }         ← h1 VE h2 (virgül = liste)

/* Pseudo-class (durum seçici) */
a:hover { }        ← Üzerine gelince
li:first-child { } ← İlk li
li:nth-child(2) { }← 2. li
p:not(.ozel) { }  ← .ozel sınıfı olmayan p'ler

/* Attribute Selector */
[type="text"] { }  ← type="text" olan input
Kural: id = sayfada tek olmalı → #id. class = birden fazla eleman paylaşabilir → .class

Pseudo Elements

Pseudo elementler (sözde elementler) HTML'e eklemeden CSS ile sanal içerik eklememizi sağlar. ::before ve ::after en çok kullanılanlar — HTML'e dokunmadan dekoratif içerik eklersin.

CSS
p::before {
  content: "→ ";    ← p'nin ÖNÜNE CSS'ten içerik ekle
  color: purple;
  /* content olmadan çalışmaz! Boş bırakmak için: content: "" */
}

p::after {
  content: " ✓";   ← p'nin SONUNA ekle
}

/* Örnek kullanım — dekoratif çizgi: */
h2::after {
  content: "";
  display: block;
  width: 40px;
  height: 3px;
  background: #7c6aff;
  margin-top: 6px;
}
Canlı Önizleme:

→ Bu paragrafın önünde ::before ile ok var ✓

Başlık h2

↑ h2::after ile dekoratif alt çizgi
Dikkat: Pseudo elementler gerçek HTML elemanı değildir. Ekran okuyucular bu içeriği okumayabilir. Sadece dekorasyon için kullan, anlamlı içerik için HTML'e yaz.
CSS Temelleri

Box Model

CSS'de her eleman bir kutudur. Bu kutunun 4 katmanı var.

content
padding
İçerik ile border arasındaki boşluk. İç boşluk.
border
Kutunun çerçevesi. Kalınlık, renk, stil belirlenebilir.
margin
Diğer elemanlarla arasındaki boşluk. Dış boşluk.
content
Asıl içerik. width/height burayı etkiler.
CSS — Box Model Kullanımı
.kutu {
  width: 200px;
  height: 100px;
  padding: 20px;         ← 4 taraf aynı
  padding: 10px 20px;    ← üst-alt / sağ-sol
  border: 2px solid red;
  margin: 16px;
  
  /* ALTIN KURAL → layout kırılmaz */
  box-sizing: border-box;
}
Dikkat: Normal halde width: 200px + padding: 20px = toplam 240px eder. box-sizing: border-box ile padding width'in içine dahil olur → toplam yine 200px.
CSS Reset'e ekle: * { box-sizing: border-box; } → bütün projeyi kurtarır.

Display Değerleri

CSS

Position Değerleri

CSS

position: relative;  ← Kendi konumuna göre offset
position: absolute;  ← En yakın relative parent'a göre
position: fixed;     ← Ekrana yapışır (sticky navbar)

/* Offset özellikleri: */

Kritik Kural: position: absolute kullanan eleman, üst elemanının position: relative olmasını bekler. Aksi halde sayfanın köküne göre konumlanır.
CSS — Absolute içinde Relative Pattern

Flexbox

CSS'in en güçlü layout aracı. Elemanları satır veya sütun halinde hizalamayı kolaylaştırır.

CSS — Flexbox Temeli
.container {
  display: flex;
  
  /* Yön */
  flex-direction: row;         ← yatay (varsayılan)
  flex-direction: column;      ← dikey
  
  /* Yatay hizalama */
  justify-content: flex-start; ← sola
  justify-content: center;     ← ortaya
  justify-content: flex-end;   ← sağa
  justify-content: space-between; ← araya boşluk
  justify-content: space-around;  ← eşit boşluk
  
  /* Dikey hizalama */
  align-items: center;        ← dikeyde ortaya
  align-items: flex-start;    ← üste
  align-items: stretch;       ← uzat (varsayılan)
  
  gap: 16px;               ← elemanlar arası boşluk
  flex-wrap: wrap;          ← sığmazsa alta geç
}

CSS Grid

Flex tek eksen (yatay veya dikey), Grid ise iki eksen aynı anda. Kompleks layoutlar için idealdir.

CSS — Grid Temeli
.grid {
  display: grid;
  
  /* Sütun tanımı */
  grid-template-columns: 200px 200px 200px; ← 3 sütun
  grid-template-columns: 1fr 1fr 1fr;       ← 3 eşit sütun
  grid-template-columns: repeat(3, 1fr);   ← aynı şey, kısa yol
  grid-template-columns: 2fr 1fr;          ← 2:1 oran
  
  /* Otomatik responsive */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  
  gap: 20px;            ← grid boşlukları
}

Responsive Tasarım

Her ekran boyutunda düzgün görünen site. Media query ile farklı ekranlara farklı CSS yazılır.

CSS — Media Query
/* Varsayılan: masaüstü */
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

/* Tablet: 768px altında */
@media (max-width: 768px) {
  .container {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Mobil: 480px altında */
@media (max-width: 480px) {
  .container {
    grid-template-columns: 1fr;  ← tek sütun
  }
}

≤ 480px

CSS Specificity

Birden fazla CSS kuralı aynı elemanı hedeflediğinde, hangisi kazanacak? Bunu specificity (özgüllük) belirler.

Kazanma Sıralaması

#id { }
100 puan
.class { }
10 puan
p { }
1 puan
* { }
0 puan
CSS — Kim Kazanır?
p { color: blue; }          ← specificity: 1
.metin { color: red; }    ← specificity: 10 → KAZANIR
#baslik { color: green; } ← specificity: 100 → EN GÜÇLÜ

/* Kombine örnek: */
div.kart p { }  ← 0+10+1+1 = 12 puan
#nav .link { }  ← 100+10 = 110 puan → kazanır
!important Kullanma: color: red !important; her şeyi ezer ama bir hata işareti. "Specificity anlamıyorum" demek gibi. Pro bu işe başvurmaz.

CSS Variables

Değerleri bir kez tanımla, her yerde kullan. Rengi değiştirmek istersen tek yerden değiştir.

CSS — Custom Properties
/* :root = tüm sayfada geçerli */
:root {
  --primary: #7c6aff;
  --secondary: #ff6a9b;
  --text: #333;
  --radius: 12px;
  --spacing: 16px;
}

/* Kullanım: */
.buton {
  background: var(--primary);
  border-radius: var(--radius);
  padding: var(--spacing);
}

.buton-ikincil {
  background: var(--secondary);
  border-radius: var(--radius);  ← Aynı radius, tek değişiklik
}
Neden Kullanmalı? Renk temasını değiştirmek istersen tek yerden değiştirirsin. Büyük projelerde hayat kurtarır.

<html lang="tr">                      ← Dil belirt, SEO için önemli

CSS — Her Projede Kullan
* {
Evrensel seçici — sayfadaki TÜM elemanları hedefler. Reset için idealdir.

  margin: 0;           ← Tarayıcı default margin'ları sıfırla
Tarayıcılar h1, p, ul gibi etiketlere varsayılan margin ekler. Bunları sıfırlayarak temiz bir temel kurarsın.

  padding: 0;          ← Default padding'leri sıfırla
Benzer şekilde varsayılan padding'leri sıfırlar. Sen ne istiyorsan onu eklersin.

  box-sizing: border-box;  ← Layout kurtarıcı — zorunlu!
padding ve border width'in içinde hesaplanır. width:200px her zaman 200px olur. Bunu eklemeden layout yapmak gereksiz acı çekmektir.

}

body {
  font-family: sans-serif;    ← Sistem sans-serif fontu
Hızlı proje için iyi başlangıç. Üretimde Google Fonts veya sistem fontlarını tercih et: font-family: 'Inter', sans-serif;

  line-height: 1.6;           ← Okunabilir satır aralığı
1.6 = font-size'ın 1.6 katı satır yüksekliği. Çok dar (1) okunmaz, çok geniş (2+) tuhaf görünür. 1.5-1.7 arası ideal.

}

Ortalama Formülü

CSS
.merkez {
  display: flex;               ← Flexbox aktif et
Bu olmadan justify-content ve align-items çalışmaz. Önce container'a flex vermek zorunlu.

  justify-content: center;   ← Yatay ortala (ana eksen)
Ana eksen (row'da yatay) boyunca hizalama. center = ortala, space-between = kenarlara yay, flex-start/end = başa/sona al.

  align-items: center;       ← Dikey ortala (çapraz eksen)
Çapraz eksen (row'da dikey) boyunca hizalama. height verilmişse dikey ortalama çalışır. height:100vh ile tam ekran ortalar.

}

Responsive Grid

CSS
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));  ← Sihirli formül
auto-fit: sığdığı kadar sütun aç. minmax(250px,1fr): her sütun en az 250px, kalan alanı paylaş. Sonuç: media query yazmadan responsive grid!

  gap: 20px;                                               ← Elemanlar arası boşluk
gap = row-gap ve column-gap'i birlikte ayarlar. Eski tarayıcılarda grid-gap kullanılırdı.

}

Navbar

HTML + CSS
<nav class="navbar">
  <a href="#" class="logo">Logo</a>
  <ul>
    <li><a href="#">Ana Sayfa</a></li>
    <li><a href="#">Hakkında</a></li>
  </ul>
</nav>

/* CSS */
.navbar {
  display: flex;                    ← Logo ve linkleri yan yana dizer
Flex olmadan logo ve ul alt alta sıralanır. Flex ile aynı satırda, hizalı olurlar.

  justify-content: space-between;  ← Logo sola, linkler sağa
space-between: elemanlar arasında eşit boşluk bırakır, uçlara yaslar. Logo sola, nav linkleri sağa gider.

  align-items: center;
  padding: 1rem 2rem;
}
.navbar ul { display: flex; list-style: none; gap: 1rem; }  ← Linkleri yatay diz
list-style:none = madde işaretlerini kaldırır. display:flex + gap ile linkler yan yana ve aralarında boşlukla dizilir.

Card Tasarımı

CSS
.card {
  background: white;
  border-radius: 12px;             ← Köşeleri yuvarla
Modern tasarımlarda genellikle 8-16px arası. Çok büyük değerler (50%) tam oval/daire yapar.

  padding: 1.5rem;
  box-shadow: 0 2px 12px rgba(0,0,0,0.08);  ← Hafif gölge
Değerler: x-offset y-offset blur-radius [spread] renk. Küçük yumuşak gölge (2px, 0.08 opacity) profesyonel görünür. Büyük gölge dramatik.

  transition: transform 0.2s;       ← Hover animasyonunu yumuşat
transform değişimini 0.2 saniyede geçişli yap. Olmadan hover anında zıplar, bununla kayar. 0.2-0.3s ideal.

}
.card:hover { transform: translateY(-4px); }  ← Yukarı kalk efekti
Hover'da kartı 4px yukarı kaydırır. "Lifting card" efekti olarak bilinir. Kullanıcıya tıklanabilir olduğunu hissettirir.

Position: Absolute İçinde Relative

CSS — En Çok Karıştırılan Pattern
.kart {
  position: relative;  ← Parent MUTLAKA relative olmalı
relative olmadan absolute child sayfa köküne (html elementine) göre konumlanır. Tamamen farklı yere gider. Bu pattern'ı ezberle: parent=relative, child=absolute.

}
.rozet {
  position: absolute;
  top: 10px; right: 10px;  ← .kart'ın sağ üstüne yapışır
top/right/bottom/left değerleri relative parent'ın kenarlarına göre hesaplanır. top:0;right:0 = sağ üst köşe.

}

Sık Yapılan Hatalar

❌ Her şey için div kullanmak → semantic kullan: header, nav, main, article, footer
❌ Inline CSS → external .css dosyası kullan. Bakımı imkansızlaşır, spesificity karışır.
❌ !important kullanmak → specificity'i anla ve düzgün yazılmış CSS'te gerek kalmaz. Kullanmak zorunda kalıyorsan kod yapısını yeniden düşün.
❌ float ile layout yapmak → 2010'ların yöntemi. Flex/grid kullan. float bugün sadece metin içinde resim sarmalamak için kullanılır.
❌ label olmadan input kullanmak → erişilebilirlik ihlali ve kötü UX. Her input'un bir label'ı olmalı.
❌ box-sizing: border-box yazmamak → * { box-sizing: border-box } her projenin 3. satırı olmalı. Yoksa padding layout'u bozar.

Genel Tablo: Bir web sitesi görüntüsü gördüğünde şu soruları sor: (1) Hangi bölümler var? → header, main, footer (2) Layout nasıl? → flex mi, grid mi? (3) Tekrar eden elemanlar var mı? → kart, buton, link → class ver. Bu sistematik yaklaşımla herhangi bir tasarımı koda dökülebilirsin.
