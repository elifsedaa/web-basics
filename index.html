HTML Nedir?

HTML (HyperText Markup Language) web sayfalarının iskeletidir. İçeriği tanımlar ama görünümü değil. Görünüm CSS'in işi.

HTML — Temel Sayfa Yapısı 
<!DOCTYPE html>          ← Tarayıcıya "HTML5 kullanıyorum" der
<html lang="tr">            ← Kök eleman, dil belirt
  <head>                      ← Görünmez meta bilgiler
    <meta charset="UTF-8">
    <title>Sayfam</title>
  </head>
  <body>                      ← Görünen her şey burada
    <h1>Merhaba Dünya!</h1>
  </body>
</html>
Altın Kural: HTML = yapı, CSS = görünüm, JavaScript = davranış. Bu ayrımı asla karıştırma.

div ve span — Fark Ne?

İkisi de anlamsız kapsayıcılardır — kendi başlarına hiçbir görsel anlam taşımazlar. Tek fark: div blok düzeyinde, span satır içi düzeyde çalışır.

HTML
<!-- DIV: block element
     → her zaman yeni satırda başlar
     → tam genişliği kaplar (width belirtmesen bile)
     → bölümleri, kartları, layout parçalarını gruplamak için -->
<div class="kart">
  <p>Kart içeriği burada.</p>
</div>
<div class="kart">
  <p>Bu div yeni satırda başladı.</p>
</div>

<!-- SPAN: inline element
     → metnin arasında akar, satır bölünmez
     → sadece içeriği kadar yer tutar
     → tek kelime / küçük parça renklendirmek için -->
<p>
  Fiyat: <span style="color:red; font-weight:bold;">299₺</span>
  — indirimli!
</p>
— div (mavi) vs span (sarı) — renk sınırları görülüyor —
DIV → tam genişliği kapladı Birinci kart içeriği
DIV → otomatik yeni satır! İkinci kart içeriği
Ürün fiyatı: 299₺ — stokta son 3 adet kaldı!
↑ SPAN: cümle akmaya devam ediyor, yeni satır açılmadı
div kullan: Bölümleri gruplamak, kart yapmak, layout parçaları. İçinde birden fazla eleman olabilir.
span kullan: Metin içinde belirli bir kelimenin rengini, boyutunu değiştirmek. Satır bölünmez.
HTML Temelleri

Semantic HTML

Semantic = anlamlı. HTML etiketleri iki gruba ayrılır: anlamsız kapsayıcılar (div, span) ve anlamlı etiketler (header, article, nav...). Anlamlı etiketler hem insanların hem de tarayıcıların kodu daha iyi anlamasını sağlar.

div Nedir, Neden Yetmez?

Bir div tarayıcıya hiçbir şey söylemez. "Bu bir şeyin etrafında bir kutu" der ama ne olduğunu söylemez. Kod büyüdükçe div dolu sayfa karmaşık hale gelir — buna "div soup" (div çorbası) denir.

HTML — div'in zayıflığı
<!-- Tarayıcı için bunların hepsi aynı: sadece bir kutu -->
<div class="header">...</div>   ← class adından anlaşılır ama tarayıcı bilmez
<div class="nav">...</div>      ← Google buranın menü olduğunu anlayamaz
<div class="article">...</div>  ← ekran okuyucu kullanıcı kaybolur
<div class="footer">...</div>  ← "div soup" — bakımı zorlaşır
❌ Div çorbası: Her şey div, anlam yok
Yanlış
<div class="header">
  <div class="nav">...</div>
</div>
<div class="main">
  <div class="post">...</div>
</div>
<div class="footer">...</div>
✅ Semantic: etiket konuşuyor
Doğru
<header>
  <nav>...</nav>
</header>
<main>

</main>

Semantic Etiket Haritası

<header> — logo, site adı, üst gezinme alanı
<nav> — navigasyon linkleri, menü
<main> — sayfanın ana içeriği (bir sayfada tek olmalı)
<section> — tematik bölüm (Hero, Özellikler, İletişim...)
<article> — bağımsız içerik (blog yazısı, haber, yorum)
<aside>
yan içerik, ilgili linkler, reklam
<footer> — alt bilgi, telif hakkı, sosyal linkler
Neden önemli? (1) Okunabilirlik — başkası koduna baktığında yapıyı anında kavrar. (2) SEO — Google, anlamlı etiketleri tarar ve içeriği daha iyi kategorize eder. (3) Erişilebilirlik — ekran okuyucular <nav>'ı görünce "burada menü var" diyerek kullanıcıyı doğru yönlendirir.

Form Yapıları

Formlar kullanıcıdan veri toplar: kayıt, giriş, iletişim, arama... Her input mutlaka bir label ile eşleşmelidir — bu hem kullanılabilirlik hem erişilebilirlik için kritik.

HTML — Eksiksiz Form Örneği
<!--
  action: form verilerinin gönderileceği adres (sunucu endpoint)
  method: GET = veriyi URL'e ekler (arama formları için)
          POST = veriyi gizli gönderir (kayıt, giriş, iletişim için)
          → şifre, kişisel veri varsa mutlaka POST
-->
<form action="/gonder" method="POST">

  <!-- TEMEL KURAL: label for="X" ↔ input id="X"
       Değerler birebir aynı olmalı!
       Bu eşleşme: label'a tıklayınca cursor o input'a gider -->
  <label for="isim">Adınız</label>
  <input
    type="text"
    id="isim"               ← label'ın for değeriyle eşleşiyor
    name="isim"             ← sunucuya bu isimle gönderilir
    placeholder="Ad Soyad"  ← ipucu metni (silinir, değer değil)
    required                ← bu alan boşsa form gönderilmez
    autocomplete="name"    ← tarayıcı otomatik doldurabilsin
  >

  <label for="email">E-posta</label>
  <input type="email" id="email" name="email" required>
  ← type="email": @ işareti yoksa submit olmaz — otomatik validate!

  <label for="mesaj">Mesajınız</label>
  <textarea
    id="mesaj"
    name="mesaj"
    rows="4"               ← görünür satır sayısı
    placeholder="Mesajınızı buraya yazın..."
  ></textarea>             ← textarea açılıp kapanır (img gibi değil!)

  <button type="submit">Gönder</button>
  ← type="submit" → formu gönderir
     type="button" → gönderMEZ, sadece JS tetikler
     type="reset"  → formu temizler (nadiren kullanılır)

</form>

CSS Temelleri

CSS Nedir?

CSS (Cascading Style Sheets) — HTML iskeletini giydiren tasarım dilidir. Renkler, boyutlar, boşluklar, fontlar, animasyonlar — bunların hepsi CSS'in işi. HTML yapıyı kurar, CSS onu güzel yapar.

Kısa Tarihçe

1994–1996 (CSS öncesi): Eskiden stil doğrudan HTML içinde <font color="red" size="4"> gibi etiketlerle veriliyordu. Her eleman için tekrar tekrar yazılıyordu. Korkunç bir dönemdi.
CSS1 → CSS2 (1996–2001): İlk CSS standartları geldi. Ama tarayıcı desteği berbattı — "Internet Explorer CSS'i" ile "gerçek CSS" ayrı şeylerdi. Geliştiriciler için kabus.
CSS3 (2010+): Flexbox, Grid, animasyonlar, değişkenler, media query'ler geldi. Bugün kullandığımız modern CSS bu. Modüler yapıda — her özellik ayrı ayrı ekleniyor.
Bugün (2024+): Container Queries, Nesting, :has() gibi güçlü özellikler geldi. Tüm modern tarayıcılar artık CSS'i tutarlı destekliyor. Altın çağ.

/* CSS yorum satırı böyle yazılır
   Tarayıcı bunu görmez, sadece sen okursun

  property: value;  /* property: value; — noktalı virgül zorunlu! */

}

CSS Nasıl Eklenir?

❌ INLINE — kaçın

Her eleman için tekrar yazılır, değişiklik tek yerden yapılamaz, bakım imkansız.

  p { color: red; }

✅ EXTERNAL — her zaman bu
<link rel="stylesheet" href="style.css">
Ayrı .css dosyası — tüm sayfalar paylaşır, değişiklik tek yerden yapılır.
CSS Temelleri

Selector'lar

CSS bir kural yazarken önce kimi hedefleyeceğini belirtmek zorunda — bu seçime selector denir. Doğru selector seçmek hem çalışır kod hem de temiz mimari için kritik.

Temel Selector'lar

CSS
/* ── TEMEL ── */
p { color: red; }        ← sayfadaki TÜM <p> etiketleri
.kart { }               ← class="kart" olan tüm elemanlar (birden fazla olabilir)
#logo { }               ← id="logo" olan sadece 1 eleman (sayfada tekrar kullanma!)
* { }                    ← evrensel: sayfadaki her şey (reset için: * { margin:0; })

/* ── KOMBİNASYON ── */
div p { }               ← div içindeki TÜM p'ler (torun dahil)
div > p { }             ← div'in sadece DOĞRUDAN çocuğu olan p'ler
h2 + p { }              ← h2'den hemen SONRA gelen kardeş p
h1, h2, h3 { }          ← virgülle liste: hepsine aynı kural (tekrar önler)

/* ── PSEUDO-CLASS (durum seçicileri) ── */
a:hover { }             ← fare üzerindeyken
a:visited { }           ← daha önce ziyaret edilmiş link
input:focus { }         ← input'a tıklanınca / tab ile gelinince
li:first-child { }      ← listenin ilk li'si
li:last-child { }       ← listenin son li'si
li:nth-child(2) { }     ← tam olarak 2. li
li:nth-child(odd) { }   ← tek numaralılar (1,3,5...) — zebra tablo için
li:nth-child(even) { }  ← çift numaralılar (2,4,6...)
p:not(.ozel) { }        ← .ozel class'ı OLMAYAN p'ler

/* ── ATTRIBUTE SELECTOr ── */
[type="text"] { }       ← type="text" olan inputlar
[href^="https"] { }     ← href "https" ile BAŞLAYAN linkler
[class*="btn"] { }      ← class içinde "btn" GEÇENler

Canlı Selector Demosu

— farklı selector'ların hedeflediği elemanlar (renk ile işaretli) —
HTML:
<ul>
  <li>Birinci</li>
  <li>İkinci</li>
  <li>Üçüncü</li>
  <li>Dördüncü</li>
  <li>Beşinci</li>

Sonuç (CSS uygulanmış):
  • Birinci ← :first-child (mavi bg)
  • İkinci ← :nth-child(even) (yeşil sol çizgi)
  • Üçüncü ← :nth-child(odd) (mor sol çizgi)
  • Dördüncü ← :nth-child(even)
  • Beşinci ← :last-child (pembe bg)
→ Birden fazla kural aynı elemana uygulanabilir (Birinci hem :first-child hem :nth-child(odd))
id vs class farkı: id = kimlik numarası gibi, sayfada tek bir elemana. class = isim gibi, aynı anda birden fazla eleman paylaşabilir. Çoğu zaman class yeterlidir — id'yi gereksiz yere kullanma.

Pseudo Elements — Sanal İçerik

::before ve ::after HTML'e dokunmadan elemana CSS'den içerik ekler. İkon, dekoratif çizgi, rozet gibi şeyler için kullanılır. content özelliği olmadan çalışmaz.

CSS
/* HTML'i değiştirmeden p'nin önüne/arkasına içerik ekle */
p::before {
  content: "→ ";         ← ZORUNLU: content olmadan çalışmaz
  color: #7c6aff;
  font-weight: bold;
}

p::after {
  content: " ✓";
  color: green;
}

/* Dekoratif çizgi örneği — content boş string olabilir */
.baslik::after {
  content: "";            ← görsel içerik yok ama display:block şart
  display: block;
  width: 40px;
  height: 3px;
  background: #7c6aff;
  margin-top: 6px;
}

/* Hover'da beliren ok — interaktif link efekti */
.link::after {
  content: " ↗";
  opacity: 0;
  transition: opacity 0.2s;
}
.link:hover::after {
  opacity: 1;             ← hover'da ok belirir
}

CSS Temelleri

Box Model

CSS'in en temel konsepti: sayfadaki her eleman bir kutudur. Bu kutu dıştan içe 4 katmandan oluşur. Bunu anlamadan layout yapmak çok zorlaşır.

content
margin — dış boşluk
Diğer elemanlarla arasındaki boşluk. Transparan — arka plan rengi buraya yayılmaz.
border — çerçeve
Kutunun görünür (ya da görünmez) çerçevesi. Kalınlık, renk ve stil ayarlanır.
padding — iç boşluk
İçerik ile border arasındaki boşluk. Arka plan rengi buraya da yayılır.
content — içerik
Asıl içerik alanı. width ve height normalde burayı etkiler.

İnteraktif Demo — Kaydır ve Gör

margin
border
padding
content
.kutu { margin: 20px; padding: 20px; border: 3px solid red; }
CSS — kısa yol yazımları
.kutu {
  /* padding kısa yolları (saat yönü: üst sağ alt sol) */
  padding: 20px;               ← 4 taraf aynı
  padding: 10px 20px;          ← üst-alt: 10px, sağ-sol: 20px
  padding: 5px 10px 15px 20px; ← üst sağ alt sol (saat yönü)
  padding-top: 10px;           ← sadece bir taraf

  /* border: kalınlık stil renk */
  border: 2px solid #333;      ← solid, dashed, dotted, double...
  border-top: 3px solid red;   ← sadece üst kenara
  border-radius: 8px;           ← köşe yuvarlatma (50% = daire)

  /* ALTIN KURAL — tüm projeye uygula */
  box-sizing: border-box;      ← padding+border, width'in içine dahil
}
box-sizing neden önemli? Normal halde: width: 300px + padding: 20px = kutunun gerçek boyutu 340px. Bu layout hesaplamalarını bozar. border-box ile padding içeriye dahil edilir, kutu hep 300px kalır.
Her projede CSS reset'ine ekle: * { box-sizing: border-box; } → tüm elemanlar tutarlı davranır, hiç pişman olmazsın.

Layout

Flexbox

CSS'in en güçlü layout aracı. Elemanları satır veya sütun halinde hizalamayı kolaylaştırır.

CSS — Flexbox Temeli
.container {
  display: flex;
  
  /* Yön */
  flex-direction: row;         ← yatay (varsayılan)
  flex-direction: column;      ← dikey
  
  /* Yatay hizalama */
  justify-content: flex-start; ← sola
  justify-content: center;     ← ortaya
  justify-content: flex-end;   ← sağa
  justify-content: space-between; ← araya boşluk
  justify-content: space-around;  ← eşit boşluk
  
  /* Dikey hizalama */
  align-items: center;        ← dikeyde ortaya
  align-items: flex-start;    ← üste
  align-items: stretch;       ← uzat (varsayılan)
  
  gap: 16px;               ← elemanlar arası boşluk
  flex-wrap: wrap;          ← sığmazsa alta geç
}

CSS Grid

Flex tek eksen (yatay veya dikey), Grid ise iki eksen aynı anda. Kompleks layoutlar için idealdir.

CSS — Grid Temeli
.grid {
  display: grid;
  
  /* Sütun tanımı */
  grid-template-columns: 200px 200px 200px; ← 3 sütun
  grid-template-columns: 1fr 1fr 1fr;       ← 3 eşit sütun
  grid-template-columns: repeat(3, 1fr);   ← aynı şey, kısa yol
  grid-template-columns: 2fr 1fr;          ← 2:1 oran
  
  /* Otomatik responsive */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  
  gap: 20px;            ← grid boşlukları
}

Responsive Tasarım

Her ekran boyutunda düzgün görünen site. Media query ile farklı ekranlara farklı CSS yazılır.

CSS — Media Query
/* Varsayılan: masaüstü */
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

/* Tablet: 768px altında */
@media (max-width: 768px) {
  .container {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Mobil: 480px altında */
@media (max-width: 480px) {
  .container {
    grid-template-columns: 1fr;  ← tek sütun
  }
}

CSS Specificity

Birden fazla CSS kuralı aynı elemanı hedeflediğinde, hangisi kazanacak? Bunu specificity (özgüllük) belirler.

Kazanma Sıralaması

#id { }
100 puan
.class { }
10 puan
p { }
1 puan
* { }
0 puan
CSS — Kim Kazanır?
p { color: blue; }          ← specificity: 1
.metin { color: red; }    ← specificity: 10 → KAZANIR
#baslik { color: green; } ← specificity: 100 → EN GÜÇLÜ

/* Kombine örnek: */
div.kart p { }  ← 0+10+1+1 = 12 puan
#nav .link { }  ← 100+10 = 110 puan → kazanır
!important Kullanma: color: red !important; her şeyi ezer ama bir hata işareti. "Specificity anlamıyorum" demek gibi. Pro bu işe başvurmaz.

CSS Variables

Değerleri bir kez tanımla, her yerde kullan. Rengi değiştirmek istersen tek yerden değiştir.

CSS — Custom Properties
/* :root = tüm sayfada geçerli */
:root {
  --primary: #7c6aff;
  --secondary: #ff6a9b;
  --text: #333;
  --radius: 12px;
  --spacing: 16px;
}

/* Kullanım: */
.buton {
  background: var(--primary);
  border-radius: var(--radius);
  padding: var(--spacing);
}

.buton-ikincil {
  background: var(--secondary);
  border-radius: var(--radius);  ← Aynı radius, tek değişiklik
}
Neden Kullanmalı? Renk temasını değiştirmek istersen tek yerden değiştirirsin. Büyük projelerde hayat kurtarır.

BEM Mimarisi

BEM = Block Element Modifier. CSS class isimlerini sistematik yazmak için bir yöntem.

CSS — BEM Yapısı
/* BLOCK — bağımsız bileşen */
.kart { }

/* ELEMENT — bloğun parçası (__) */
.kart__baslik { }
.kart__resim { }
.kart__icerik { }

/* MODIFIER — varyasyon (--) */
.kart--buyuk { }
.kart--aktif { }
.kart--disabled { }
HTML — BEM Kullanımı
<div class="kart kart--buyuk">
  <img class="kart__resim" src="...">
  <h2 class="kart__baslik">Başlık</h2>
  <p class="kart__icerik">İçerik</p>
</div>

.kart-buyuk-baslik
.red-button-hover
.mystylee123
✅ BEM ile:
.kart--buyuk
.buton--kirmizi
.buton__ikon
